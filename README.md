# Локальный ИИ-ассистент с загрузкой файлов

Учебный мини-проект: вы загружаете свои файлы (документы или картинки),
задаёте вопрос — и **локально установленная модель Ollama** отвечает по их содержимому.
Никакие данные не уходят в облако — всё работает на вашем компьютере.

Это пример **RAG** (Retrieval-Augmented Generation, «генерация с поиском по документам»):
текст режется на фрагменты, по вопросу ищутся самые подходящие, и модель отвечает,
опираясь на них. Всё приложение — в одном файле `app.py`.

---

## Что нужно установить заранее (один раз)

| Что | Зачем | Где взять |
|-----|-------|-----------|
| **Python 3.9+** | на нём написано приложение | https://www.python.org/downloads/ (Windows: галочка *Add Python to PATH*) |
| **Ollama** | локальный «движок» нейросетей | https://ollama.com/download |

> Модель скачивать вручную не обязательно — скрипт запуска предложит сделать это сам.

---

## Запуск одной командой

Скрипт сам создаст окружение, поставит зависимости, проверит Ollama и
**предложит скачать модель**, если её ещё нет.

**macOS / Linux:**

```bash
bash run.sh
```

**Windows:** дважды кликните по файлу **`run.bat`** (или в консоли — `run.bat`).

После запуска приложение откроется в браузере: **http://localhost:8501**

> Первый запуск дольше: ставятся зависимости и скачивается модель (~2 ГБ).
> Следующие запуски — за секунды.

<details>
<summary>Запустить вручную (без скрипта)</summary>

```bash
ollama pull llama3.2            # текстовая модель
ollama pull nomic-embed-text   # поиск по файлам
pip install -r requirements.txt
streamlit run app.py
```
</details>

## Первое задание (проверка, что всё работает)

1. Запустите приложение (см. выше).
2. Слева на вкладке **«Файлы»** загрузите готовый пример `sample_docs/regulation.txt`.
3. Спросите в чате: **«Какая гарантия на оборудование?»**
4. Модель должна ответить **«24 месяца»** и сослаться на файл `[regulation.txt]`.

Получилось — значит, окружение настроено правильно. Дальше пробуйте свои файлы.

## Что умеет

- Загрузка нескольких файлов: `txt, md, csv, pdf, docx` и картинки `png, jpg, webp`
- Семантический RAG: поиск нужных фрагментов по **векторным эмбеддингам** (с откатом к поиску по словам)
- Картинки уходят в vision-модель (`gemma3` / `llava`)
- Чат с историей и потоковым ответом (текст «печатается» на лету)
- Выбор модели и температуры в боковой панели
- Всё локально — приватность по умолчанию

## Если что-то пошло не так

| Симптом | Что сделать |
|---------|-------------|
| **«Нет связи с Ollama»** в приложении | Откройте терминал и выполните `ollama serve`. Затем перезапустите. |
| **«Модель не установлена»** | `ollama pull llama3.2` (и `ollama pull nomic-embed-text` для поиска). |
| `python: command not found` (Windows) | Переустановите Python с галочкой *Add Python to PATH*. |
| Картинки анализируются плохо | Поставьте vision-модель: `ollama pull gemma3` или `ollama pull llava`. |
| Хочу всё начать с нуля | Удалите папку `.venv` и запустите `run.sh` / `run.bat` снова. |

## Ollama на другом компьютере (по желанию)

Если Ollama стоит на отдельном сервере, укажите его адрес перед запуском:

```bash
OLLAMA_URL="http://10.50.50.202:11434" bash run.sh
```

На сервере Ollama должен слушать сеть: `OLLAMA_HOST=0.0.0.0:11434 ollama serve`.
