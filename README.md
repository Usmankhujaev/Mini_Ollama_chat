# 🤖 Локальный ИИ-ассистент с загрузкой файлов

Мини-чатбот: студент загружает свои файлы (документы или картинки),
задаёт вопрос — и **локально установленный Ollama** отвечает по содержимому файлов.
Никакие данные не уходят в облако.

Концепция взята из практикума **Уроков 3–4** (локальный Ollama API) и
повторяет идею «файлы → контекст → ответ модели» из проекта *dbp-assistant*,
но без базы данных и сложной инфраструктуры — всё в одном файле `app.py`.

---

## 🚀 Запуск одной командой

**Предусловие:** установлен и запущен [Ollama](https://ollama.com), скачана хотя бы одна модель:

```bash
ollama pull llama3.2        # текстовая модель
ollama pull gemma3          # умеет работать и с картинками (vision)
ollama pull nomic-embed-text  # эмбеддинги для семантического поиска по файлам
```

**macOS / Linux:**

```bash
bash run.sh
```

**Windows:**

```bat
run.bat
```

Скрипт сам создаст виртуальное окружение, поставит зависимости и откроет приложение
в браузере: **http://localhost:8501**

> Хотите вручную? `pip install -r requirements.txt && streamlit run app.py`

---

## 🖥 Удалённый сервер Ollama

Если Ollama стоит на другом компьютере/сервере, задайте его адрес перед запуском:

```bash
OLLAMA_URL="http://10.50.50.202:11434" bash run.sh
```

На сервере Ollama должен слушать сеть:  `OLLAMA_HOST=0.0.0.0:11434 ollama serve`
(и порт 11434 открыт в фаерволе).

## 📋 Что умеет

- 📎 Загрузка нескольких файлов: `txt, md, csv, pdf, docx` и картинки `png, jpg, webp`
- 🔎 Семантический RAG: текст режется на чанки, релевантные ищутся по **векторным эмбеддингам** (косинусная близость), с откатом к пословному поиску
- 🖼 Картинки уходят в vision-модель (по умолчанию `gemma3` / `llava`)
- 💬 Чат с историей и потоковым ответом (как «печатает» ChatGPT)
- ⚙️ Выбор модели и температуры в боковой панели
- 🔒 Всё локально — приватность по умолчанию

Готовый пример для теста лежит в `sample_docs/regulation.txt` — загрузите его и
спросите, например: *«Какая гарантия на оборудование?»*

## 🧩 Куда расти (для занятий)

- Хранить эмбеддинги в векторной БД (pgvector / FAISS) вместо пересчёта в памяти — масштабирование на большие корпусы.
- Сохранять историю и файлы в БД (как в dbp-assistant: PostgreSQL + pgvector).
- Добавить цитаты-источники к ответу и оценку уверенности.
